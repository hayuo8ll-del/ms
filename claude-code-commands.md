# Claude Code コマンド一覧

Claude Code（CLI）で使えるコマンド・オプション・ショートカットをカテゴリ別にまとめた一覧です。

## 1. スラッシュコマンド

対話セッション内で `/コマンド名` の形式で実行します。

### 初期化・管理

| コマンド | 説明 |
|---|---|
| `/init` | プロジェクトを解析し `CLAUDE.md` を初期化する |
| `/memory` | メモ（記憶しておきたい情報）を管理する |
| `/cd` | 作業ディレクトリを移動する |

### モデル・設定

| コマンド | 説明 |
|---|---|
| `/model` | 使用するモデルを切り替える |
| `/config` | 各種設定を変更する |
| `/effort` | 思考の労力レベルを変更する（低 〜 max） |

### コンテキスト管理

| コマンド | 説明 |
|---|---|
| `/context` | 現在のコンテキスト使用状況を表示する |
| `/compact` | これまでの会話を要約してコンテキストを圧縮する |

### コード作業

| コマンド | 説明 |
|---|---|
| `/plan` | 実装計画を立てるモードに入る |
| `/diff` | 現在の差分を表示する |
| `/code-review` | 現在の差分をレビューする |
| `/security-review` | 現在のブランチの変更をセキュリティ観点でレビューする |

### セッション管理

| コマンド | 説明 |
|---|---|
| `/clear` | 会話履歴をクリアする |
| `/resume` | 過去のセッションを再開する |
| `/branch` | 別ブランチとして分岐する |
| `/fork` | セッションを複製する |
| `/rewind` | 過去の状態に巻き戻す |

### 並列・バックグラウンド処理

| コマンド | 説明 |
|---|---|
| `/batch` | 大規模な変更をバッチ処理する |
| `/background` | タスクをバックグラウンド実行に切り替える |

### ツール連携

| コマンド | 説明 |
|---|---|
| `/mcp` | MCP（外部ツール）接続を管理する |
| `/debug` | デバッグ情報を表示する |
| `/doctor` | 環境の問題を診断・修復する |

### その他

| コマンド | 説明 |
|---|---|
| `/copy` | 直前の出力をコピーする |
| `/export` | 会話をファイルにエクスポートする |
| `/help` | ヘルプを表示する |
| `/exit` | セッションを終了する |

## 2. CLIオプション（起動時フラグ）

ターミナルで `claude [オプション]` の形式で使用します。

### セッション制御

| オプション | 説明 |
|---|---|
| `-p` | 対話なしで一度だけ実行し、結果を出力する |
| `-c` | 直前のセッションを継続する |
| `-r`, `--resume` | 過去のセッションを再開する |
| `-n` | セッションに名前を付ける |
| `--bg` | バックグラウンドで実行する |

### モデル関連

| オプション | 説明 |
|---|---|
| `--model` | 使用モデルを指定する |
| `--effort` | 思考の労力レベルを指定する |
| `--advisor` | アドバイザーモデルを指定する |
| `--fallback-model` | 主モデル失敗時のフォールバックモデルを指定する |

### パーミッション

| オプション | 説明 |
|---|---|
| `--permission-mode` | パーミッションモードを指定する |
| `--dangerously-skip-permissions` | 権限確認をスキップする（注意：危険） |
| `--safe-mode` | 安全モードで起動する |

### 入出力

| オプション | 説明 |
|---|---|
| `--output-format` | 出力形式を指定する（例: json） |
| `--json-schema` | JSON出力のスキーマを指定する |

### カスタマイズ

| オプション | 説明 |
|---|---|
| `--system-prompt` | システムプロンプトを指定する |
| `--settings` | 設定ファイルを指定する |
| `--add-dir` | 追加の作業ディレクトリを指定する |
| `--mcp-config` | MCP設定ファイルを指定する |

### 制限・診断

| オプション | 説明 |
|---|---|
| `--max-turns` | 最大ターン数を制限する |
| `--max-budget-usd` | 予算上限（USD）を指定する |
| `--tools` | 使用可能なツールを制限する |
| `--bare` | 最小構成で起動する |
| `--debug` | デバッグモードで起動する |
| `--version` | バージョンを表示する |

## 3. キーボードショートカット

| ショートカット | 説明 |
|---|---|
| `Ctrl+C` | 実行中の処理を中断する |
| `Ctrl+D` | セッションを終了する |
| `Ctrl+O` | トランスクリプト（詳細ログ）を表示する |
| `Ctrl+T` | To-Doリストを表示する |
| `Enter` | メッセージを送信する |
| `Ctrl+J` | 改行を挿入する |
| `Ctrl+L` | 画面を再描画する |
| `Ctrl+S` | 変更をstashする |
| `Meta+P` | モデルピッカーを開く |
| `Meta+O` | 高速モードを切り替える |
| `Ctrl+R` | 履歴を検索する |
| `↑` / `↓` | 入力履歴を移動する |
| `Ctrl+G` / `Ctrl+X Ctrl+E` | 外部エディタを開く |
| `Ctrl+V` | 画像を貼り付ける |
| `Ctrl+_` | 直前の操作を元に戻す |

キーバインドは `~/.claude/keybindings.json` でカスタマイズ可能です。

## 4. 設定ファイルの階層

優先度が高い順に以下のスコープで設定が適用されます（上ほど優先）。

| スコープ | パス | Git共有 |
|---|---|---|
| Managed（組織管理） | 管理者が配布する設定 | - |
| User（ユーザー） | `~/.claude/settings.json` | されない |
| Project（プロジェクト） | `.claude/settings.json` | される |
| Local（ローカル） | `.claude/settings.local.json` | されない（`.gitignore` 対象） |

設定変更は自動的に反映されます。現在の設定状況は `/status` コマンドで確認できます。
